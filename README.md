## 🛠️ Project Overview & Scope

*   **Test Cases:** 220+ comprehensive manual test cases covering functional, UI/UX, and security aspects.
*   **Platforms/Browsers:** Tested on Chrome, Edge, and Firefox (Desktop).
*   **Target Application:** [OpenCart Demo Website](https://demo.opencart.com/)

### 🔍 Key Findings & Logged Bugs
*   **Security:** Identified critical vulnerabilities in password complexity and account security.
*   **Functional:** Found issues with input validation in Registration and Cart calculations.
*   **UI/UX:** Documented missing success feedback on critical user actions.

## ⚙️ Repository Architecture & Automation
Even though this is primarily a manual testing project, it utilizes a modern **Automated Version Control** workflow to back up and track changes:

1. **Test Management:** Maintained seamlessly on Google Sheets for real-time editing and premium formatting.
2. **Data Pipeline:** A custom Python script (`update_tests.py`) automatically fetches the latest data from the Google Sheet API, exports it as a `.csv` file, and pushes it directly to this repository.
3. **Version Tracking:** GitHub acts as a secure change-log provider, keeping track of every single update over time.
